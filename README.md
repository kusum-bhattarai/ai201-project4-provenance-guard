# Provenance Guard

A backend system a creative-sharing platform can plug in to answer one question about
submitted text: **was this written by a person, or generated by AI?** It classifies
content with two independent signals, scores its confidence honestly (it says "I'm not
sure" instead of guessing), shows readers a plain-language transparency label, and gives
creators a way to appeal.

Design north star: **on a writing platform, wrongly branding a human's work as AI is the
worst outcome.** Every scoring and label decision below is biased against that false
positive.

> Full pre-implementation design is in [`planning.md`](./planning.md).

## Architecture overview — the path a submission takes

```
POST /submit {text, creator_id}
      │
      ├─▶ Signal 1: Groq LLM (llama-3.3-70b-versatile)  ── llm_score ∈ [0,1] ─┐  (semantic)
      ├─▶ Signal 2: Stylometric heuristics (pure Python) ── style_score ∈ [0,1]┤  (structural)
      │                                                                        ▼
      │                                        Confidence scorer:
      │                                        base = 0.6·llm + 0.4·style
      │                                        confidence = 0.5 + (base−0.5)·(1 − 0.5·|llm−style|)
      │                                        → attribution band (ai / uncertain / human)
      │                                                                        │
      │                                        Label generator ── label text ──┤
      │                                                                        ▼
      └──────────────────── structured row ──────────────────────────▶  Audit log (SQLite)
                                                                                 │
      response {content_id, attribution, confidence, signals, label} ◀──────────┘

POST /appeal {content_id, creator_reasoning}
      └─▶ look up original decision ─▶ status = under_review ─▶ new audit row (links original
          scores + appeal reasoning) ─▶ response {status: under_review}
```

A creator submits text. It is scored by **two independent signals** — one that reads the
text semantically (an LLM) and one that measures its structure statistically (stylometry).
The two scores are combined into a single confidence = **P(AI)**, with a disagreement
penalty that pulls the result toward "uncertain" whenever the signals conflict. The
confidence maps to a three-band attribution and a plain-language transparency label.
Everything — both signal scores, the combined confidence, the verdict, the label — is
written to a structured SQLite audit log. An appeal looks up the original decision, flips
the content's status to `under_review`, and logs the creator's reasoning next to the
original scores for a human reviewer.

## Detection signals

Two signals that capture **genuinely different properties** of the text, so the pair is
more informative than either alone.

### Signal 1 — LLM classifier (Groq `llama-3.3-70b-versatile`) · *semantic*
- **Measures:** holistic "does this read human?" — tone, idiom, argumentative flow, the
  presence or absence of lived-in specificity. Returns `ai_probability ∈ [0,1]` + a
  one-sentence rationale (structured JSON, `temperature=0`).
- **Why I chose it:** AI prose has a recognizable gestalt — smooth, evenly hedged,
  topic-complete. An LLM is good at feeling that out in a way no formula can.
- **What it misses:** it can be confidently wrong and is non-deterministic. It
  **false-positives on formal, even human prose** (academic writing, non-native-English
  writers who learned in a formal register) and can be fooled by lightly-edited AI text.
  It offers a vibe with a number, not statistical evidence.

### Signal 2 — Stylometric heuristics (pure Python) · *structural*
- **Measures:** how *uniform* the text is, via three sub-metrics combined into
  `style_score ∈ [0,1]` (1 = uniform / AI-like):
  - **Burstiness** — coefficient of variation of sentence lengths. Humans vary sentence
    length a lot; AI is even. (weight 0.5)
  - **Type-token ratio** — vocabulary diversity over a capped 200-word window; heavy
    repetition / low diversity reads uniform. (weight 0.2)
  - **Human-punctuation density** — em/en dashes, ellipses, parentheses, semicolons,
    standalone lowercase "i"; more of these reads human. (weight 0.3)
  - Very short text (< 40 words) is damped toward 0.5 because its statistics are unstable.
- **Why I chose it:** it's statistical, content-blind, and cheap — its failure modes barely
  overlap with the LLM's. Where the LLM is fooled by *formal* human prose, stylometry can
  cast a dissenting vote (and vice versa).
- **What it misses:** it's genre-blind. A repetition-heavy human poem looks "uniform" =
  AI; casual AI output prompted to sound messy looks bursty = human. It knows nothing about
  meaning.

**Why the pairing works:** one signal is semantic, the other structural. Their blind spots
don't line up, so their **disagreement is itself a signal of genuine uncertainty** — which
the scorer uses directly (below).

## Confidence scoring

Confidence is a single number: **P(AI-generated) ∈ [0,1]**. This makes uncertainty
intrinsic — `0.5` is maximum "I can't tell," `0.95` is a confident AI verdict, `0.05` is a
confident human verdict. A `0.51` and a `0.95` are meaningfully different outputs, not a
binary flip.

```
base         = 0.6 · llm_score + 0.4 · style_score        # LLM weighted higher
disagreement = |llm_score − style_score|
confidence   = 0.5 + (base − 0.5) · (1 − 0.5 · disagreement)
```

The **disagreement shrinkage** is the heart of the false-positive protection: when the two
signals conflict, the score is pulled toward 0.5 (uncertain), so **no single confident
signal can brand content on its own.** The LLM is weighted higher (0.6) because it captures
meaning — but it's exactly the signal that false-positives on formal human writing, so
stylometry's dissent is allowed to blunt it.

### Attribution bands (asymmetric — biased against false positives)

| P(AI) | Attribution | Label |
|-------|-------------|-------|
| `≥ 0.70` | `likely_ai` | high-confidence AI |
| `0.35 – 0.70` | `uncertain` | uncertain |
| `≤ 0.35` | `likely_human` | high-confidence human |

The AI threshold (0.70) sits far above the human threshold's mirror image: I demand
*strong* evidence before telling a reader something was AI-made, and give a wide benefit of
the doubt to the human/uncertain zone.

### How I validated it's meaningful

I ran six deliberately chosen inputs through the full pipeline and inspected both signal
scores plus the combined result. Scores vary across the range and land in the intended
bands:

| Input | llm | style | **confidence** | verdict |
|-------|-----|-------|----------------|---------|
| Uniform AI listicle (both signals agree) | 0.90 | 0.65 | **0.76** | `likely_ai` |
| Formal AI paragraph (stylometry dissents) | 0.90 | 0.48 | **0.69** | `uncertain` |
| Formal human econ writing | 0.80 | 0.59 | **0.69** | `uncertain` |
| Lightly-edited AI (remote-work) | 0.70 | 0.37 | **0.56** | `uncertain` |
| Repetition-heavy human poem | 0.20 | 1.00 | **0.51** | `uncertain` |
| Casual human (ramen review) | 0.20 | 0.01 | **0.16** | `likely_human` |

Two things I specifically checked for:
- **Clearly-different inputs produce clearly-different scores** (0.76 vs 0.16), and only
  band-crossing when the evidence warrants it.
- **The disagreement penalty protects the exact cases it should.** The *repetition-heavy
  human poem* is the payoff: stylometry false-positives it hard (`style = 1.00`, pure
  repetition), but the LLM dissents (`0.20`), the disagreement is huge (`0.80`), and the
  score is dragged to `0.51` = **uncertain — not a false AI brand.**

**High-confidence** (`likely_ai`, confidence **0.76**):


> honest "uncertain" middle.

## Transparency label

Three variants, chosen by confidence band. They are plain-language, non-technical, and
non-accusatory; each states that the verdict is an automated estimate, and the AI variant
points to the appeal path. The displayed percentage is always framed in the *direction of
the verdict* so a reader never has to mentally invert a probability.

**High-confidence AI** (`confidence ≥ 0.70`):
> 🤖 **Likely AI-generated**
> Our automated analysis suggests this content was most likely created with the help of
> generative AI (estimated {P_ai}% likelihood). This is an automated estimate, not a
> certainty. If you're the creator and believe this is wrong, you can appeal this result.

**High-confidence human** (`confidence ≤ 0.35`):
> ✍️ **Likely human-written**
> Our automated analysis found no strong signs of AI generation in this content (estimated
> {P_human}% likelihood it's human-written). This is an automated estimate and not a
> guarantee of authorship.

**Uncertain** (`0.35 < confidence < 0.70`):
> ❓ **Not enough signal to tell**
> Our automated analysis couldn't confidently determine whether a person wrote this or it
> was generated with AI. Rather than risk mislabeling the creator's work, we're showing
> this note instead of a verdict.

`{P_ai}` = `round(confidence·100)`; `{P_human}` = `round((1−confidence)·100)`. The design
respects the false-positive asymmetry: the human and uncertain variants are reassuring and
never accuse; only the AI variant makes a claim, and even that one is hedged and appealable.

## Appeals workflow

A creator who disputes a verdict sends `POST /appeal` with the `content_id` (from their
`/submit` response) and free-text `creator_reasoning`. The system:

1. Looks up the original decision (404 if the `content_id` is unknown).
2. Sets the content's status to **`under_review`**.
3. Writes a **new audit row** that links the appeal to the original decision — preserving
   the original `attribution`, `confidence`, `llm_score`, and `style_score`, and recording
   the `appeal_reasoning`.
4. Returns a confirmation.

No automated re-classification. A human reviewer opening the queue (`status = under_review`)
sees the original text, both signal scores, the combined verdict, and the creator's reasoning
side by side — enough to make a manual call.

```bash
curl -s -X POST http://127.0.0.1:5001/appeal \
  -H "Content-Type: application/json" \
  -d '{"content_id": "614a6403-3b5e-41b7-851a-30f21e2d4222",
       "creator_reasoning": "I wrote this myself; English is my second language so it reads formal."}'
# → {"content_id": "...", "status": "under_review",
#    "message": "Your appeal has been received and this content is now under review."}
```

## Rate limiting

`/submit` is limited with Flask-Limiter (in-memory storage) to:

```
@limiter.limit("10 per minute;100 per hour")
```

**Reasoning for these specific values:**
- **10 / minute** — a real creator submits their own work occasionally, not in bursts. Ten
  a minute is generous headroom for someone iterating on a draft or checking a few pieces,
  while a script trying to brute-force the classifier (or run up Groq API cost) hits the
  wall almost immediately.
- **100 / hour** — a sane sustained ceiling for one creator. Even a very active writer won't
  legitimately submit 100 distinct pieces an hour; a sustained flood attempt is throttled
  well before it becomes expensive or a denial-of-service risk.

The limit is per client IP (`get_remote_address`). It's a deliberately tight pairing:
the per-minute limit stops bursts, the per-hour limit stops slow-drip flooding.

**Evidence** — 12 rapid requests against `/submit` (first 10 succeed, the rest are blocked):

via `GET /log`. Each row carries the timestamp, content/creator IDs, the event type, the
attribution, the combined confidence, **both individual signal scores**, the status, the
label shown, and (for appeals) the creator's reasoning. Classification rows also stash the
LLM rationale and full stylometry metrics in an `extra` JSON blob.

## Known limitations

The system will **misclassify repetition-heavy or structurally-simple human creative
writing** — villanelles, refrains, nursery-style rhymes, list poems, minimalist prose.
This is a direct consequence of the stylometry signal: deliberate repetition collapses the
type-token ratio and flattens sentence-length variance, which are precisely the features
the signal reads as "uniform = AI." My test poem ("Do not go gentle…") scored `style = 1.00`
— the maximum AI reading — despite being famous human verse.

The disagreement penalty rescues many of these cases (the LLM usually recognizes the poetic
voice and dissents, dragging the result to *uncertain* rather than *AI* — exactly what
happened with the test poem, final confidence `0.51`). But it's a genuine weak spot: a poem
where the LLM *also* leans AI would slip through, and even the "uncertain" verdict is a poor
experience for a creator who knows they wrote it. This is why the appeal path exists.

A second known-weak case: **very short submissions** (a haiku, a one-line caption). Under 40
words, stylometry's statistics are dominated by one or two sentences and the type-token ratio
trends mechanically toward 1.0. I damp the stylometry signal toward 0.5 for short inputs so it
can't confidently mislabel them, which means the system leans *uncertain* on short text rather
than trusting numbers it can't compute reliably.

## Spec reflection

**One way the spec helped:** writing the confidence formula *and its exact thresholds* into
`planning.md` before coding turned scoring from a fuzzy intention into something I could
verify line-by-line. When I generated the scorer I diffed it against the written
`base / disagreement / shrinkage` formula and the `0.70 / 0.35` bands and caught nothing
silently drifting — and the calibration run (six inputs) confirmed the bands behaved without
me guessing at magic numbers.

**One way the implementation diverged:** my planning table implied a "clearly AI" text would
land in `likely_ai`. In practice the canonical *formal* AI paragraph only reached `0.69`
(uncertain), because stylometry rated its diverse vocabulary as human-like and the
disagreement penalty held it back — only a *structurally uniform* AI text (the listicle)
actually cleared the AI band. I chose to **keep** this conservative behavior rather than tune
it away: landing a stylometrically-ambiguous AI paragraph in "uncertain" is the honest,
false-positive-averse outcome, even though it wasn't what my spec table first suggested.

## AI usage

**1. Generating the stylometry signal from the sub-metric spec.** I directed the AI tool to
implement `stylometry_signal(text)` from my planning.md sub-metrics (burstiness, TTR,
punctuation density) with a `[0,1]` output. It produced a reasonable first version, but I
**added the short-text damping** (`n_words < 40` → shrink toward 0.5): the first version
returned confident scores for two-sentence inputs where the type-token ratio is mechanically
~1.0, which would have produced unstable verdicts on captions and haiku. I also **re-derived
the TTR mapping** so low diversity (heavy repetition) reads as uniform, to match how the
signal behaves on my poem edge case.

**2. Generating the Groq LLM signal.** I directed the AI to write `llm_signal(text)` calling
`llama-3.3-70b-versatile` with a structured-JSON response format. I **overrode** two things:
I set `temperature=0` for repeatable scores, and I **added graceful degradation** — on any
API or JSON-parse failure the function returns a neutral `0.5` with an error rationale
instead of raising, so a flaky API response can never take down the whole submission
pipeline. I also rewrote the system prompt to spell out the specific human/AI tells I wanted
weighed, rather than leaving the judgment vague.
